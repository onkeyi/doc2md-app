Features

Everything you need,
nothing you don't.

doc2md handles the messy parts of document conversion so you get clean, portable Markdown every time.

🖥️
Native macOS App

Drag-and-drop GUI built with Tauri. Lightweight, fast, and feels at home on macOS — no Electron bloat.

⌨️
Powerful CLI

Integrate into scripts, CI pipelines, or any workflow. One command converts a single file or an entire folder.

📁
Batch Processing

Convert dozens of documents at once. The app queues files automatically and shows per-file progress.

🖼️
Image Extraction

Embedded images are extracted and saved alongside the Markdown, with correct relative links in the output.

📊
Excel Sheet Split

Multi-sheet workbooks are split into one Markdown file per sheet, keeping tables readable and structured.

🔒
Fully Offline

No internet connection required. Your documents never leave your machine — complete privacy by design.

How it works

Three steps to clean Markdown.

No setup, no accounts, no configuration files. Open the app and start converting.

01
Drop your files

Drag .docx or .xlsx files into the app, or browse for them. You can also point the CLI at a folder for bulk conversion.

→
02
Hit convert

Choose your output directory and click Convert. The Rust core processes each file in milliseconds, extracting text, tables, and images.

→
03
Get clean Markdown

Your .md files are ready. Headings, lists, code blocks, tables, and image references — all faithfully converted.

CLI

Built for scripts
and pipelines.

The doc2md CLI is installed automatically when you first launch the macOS app. Use it everywhere.

✓ Single file or folder — convert one document or an entire directory tree
✓ Configurable output path — write alongside source or to a specified destination
✓ CI-friendly exit codes — exits 0 on success, non-zero on any failure
✓ Auto-installed with the app — no separate package manager needed
shell
# Convert a single file
doc2md report.docx

# Specify output directory
doc2md report.docx -o ./output/

# Batch convert a folder
doc2md ./documents/

# Pipe into another tool
doc2md notes.docx --stdout | pandoc -o notes.html
